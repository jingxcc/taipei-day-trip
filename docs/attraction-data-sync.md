# Attraction Data Sync

This document describes how to synchronize the latest Taipei Open API attraction data with the project database.

For mapping definitions, see [attraction-mapping-rules.md](./attraction-mapping-rules.md).

## Synchronization Flow

```md
Taipei Open API
│
▼
compare_attractions.py
│
▼
attraction-manual-mapping.json
│
├── generate_attraction_update_sql.py
└── generate_attraction_insert_sql.py
│
▼
MySQL
```

## Update Steps

1. Export a database backup.
2. Execute `add_is_active.sql`.
3. Review the generated `update_attractions.sql`.
4. Execute `update_attractions.sql`.
5. Execute `remove_legacy_image_urls.sql` to remove obsolete image URLs from the legacy Taipei Open API.
6. Review the generated `insert_attractions.sql`.
7. Execute `insert_attractions.sql`.
8. Verify the updated data.

## Verification

### Verify inactive attractions

```sql
SELECT id, attraction_name
FROM attraction
WHERE is_active = FALSE;
```

### Verify legacy image URLs are removed

```sql
SELECT *
FROM image_url
WHERE url LIKE "%d_upload_ttn%";

SELECT DISTINCT
    a.id,
    a.attraction_name,
    a.is_active
FROM image_url AS iu
LEFT JOIN attraction AS a
    ON iu.attraction_id = a.id
WHERE iu.url LIKE "%d_upload_ttn%";
```

### Verify image counts

```sql
SELECT
    attraction_id,
    attraction_name,
    COUNT(*)
FROM image_url AS iu
LEFT JOIN attraction AS a
    ON iu.attraction_id = a.id
GROUP BY attraction_id;
```

### Verify inserted attractions

```sql
SELECT
    a.attraction_name,
    a.address,
    a.transport,
    a.is_active,
    m.mrt_name
FROM attraction a
LEFT JOIN attraction_mrt am
    ON a.id = am.attraction_id
LEFT JOIN mrt m
    ON am.mrt_id = m.id
WHERE a.attraction_name IN (
    ...
);
```

## Notes

- `update_attractions.sql` is generated by `generate_attraction_update_sql.py`.
- `insert_attractions.sql` is generated by `generate_attraction_insert_sql.py`.
- `remove_legacy_image_urls.sql` removes obsolete image URLs from the legacy Taipei Open API.

### Synchronization Rules

- Address, latitude, longitude, and images are synchronized from the latest Taipei Open API.
- Category is **not** synchronized with the latest Taipei Open API. Existing category assignments are preserved.
- The latest Taipei Open API no longer provides `direction` or `MRT` information. Existing values are preserved for updated attractions.
- For `add` mappings, `category`, `transport` (`direction`), and `MRT` must be specified manually.
