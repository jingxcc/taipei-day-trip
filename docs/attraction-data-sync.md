# Attraction Data Sync

This document describes how to synchronize the latest Taipei Open API attraction data with the project database.

## Synchronization Flow

```md
Taipei Open API
│
▼
compare_attractions.py
│
▼
attraction-manual-mapping.json
│
├── generate_attraction_update_sql.py
└── generate_attraction_insert_sql.py
│
▼
MySQL
```

## Update Steps

1. Export a database backup.
2. Execute `add_is_active.sql`.
3. Review the generated `update_attractions.sql`.
4. Execute `update_attractions.sql`.
5. Execute `remove_legacy_image_urls.sql`.
6. Verify the updated data.

## Verification

### Verify inactive attractions

```
SELECT id, attraction_name
FROM attraction
WHERE is_active = FALSE;
```

## Notes

- `update_attractions.sql` is generated by `generate_attraction_update_sql.py`.
- `remove_legacy_image_urls.sql` removes obsolete image URLs from the legacy Taipei Open API.
- Split attractions are deactivated by `generate_attraction_update_sql.py` and inserted by `generate_attraction_insert_sql.py`.
